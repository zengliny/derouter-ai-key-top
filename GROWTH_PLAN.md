**目标：** 快速获客 + 长期稳定运营  
**适用平台：** 微信、小红书、抖音、知乎、B 站、V2EX  
**日期：** 2026-04-08

---

## ⚠️ 重要提醒：合规第一

**API 中转服务的敏感点：**
1. 涉及"翻墙"、"代理"等关键词容易被封
2. 涉及"低价"、"返利"可能被判定为灰产
3. 直接放外链容易被平台限流

**核心策略：** 用"技术分享"包装，用"私域流量"转化

## 二、防封号核心技巧

### 1. 内容包装策略

| 真实意图 | 安全表达 |
|---------|---------|
| "翻墙使用" | "国内可用" |
| "API 中转" | "API 聚合服务" |
| "低价充值" | "团购优惠" |
| "代理服务" | "技术服务" |

### 2. 链接处理

```
❌ 危险做法：
直接放 derouter.ai-key.top

✅ 安全做法：
- 用短链接（oris.in/t.cn）
- 用二维码（微信里）
- 引导私信获取
- 用网盘链接包装
```

### 3. 账号矩阵

```
为什么要矩阵？
- 单号风险集中
- 内容可以相互引流
- 封了有备用

建议配置：
- 小红书：3-5 个号
- 抖音：3-5 个号
- 知乎：2-3 个号
- 微信：1 个企业号 +2 个个人号
```

### 4. 敏感词规避

**高危词（慎用）：**
```
翻墙、科学上网、VPN、梯子、
代理、中转、低价、充值、返利
```

**安全词（可用）：**
```
国内可用、免翻、直连、
技术服务、团购优惠、优惠价
```

```

```
日收入 = 流量 × 转化率 × 客单价

- 日引流 100 人
- 注册转化 30% = 30 人
